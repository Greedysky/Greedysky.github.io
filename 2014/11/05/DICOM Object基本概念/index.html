DICOM Object基本概念

技术专帖
  • C++
  • C语言
  • DICOM

DICOM Object组成

       IOD(Information Object Definition)

       影像物件(IOD)由 影像属性(Attribute)和象素信息(Pixel data)组成。

影像属性(Attribute)

       一个影像物件包含多个DICOM attribute

       一个DICOM attribute包含一个Data element，多个data element按照顺序进行传输组成了一个data set Data element的组成为：Tag + (VR) + Value Length + Value field

       如下图示：

       1)Tag位介绍：The Identifier of the attribute- 属性识别

       例：(0010,0020)是一个标准的Tag位表示，其中：0010为group number，0020为element number，同group的还有(0010,0010),(0010,0030)等Tag的定义在第6章Data Dictionary中可以查到

       2)VR介绍：The Data type of the attribute—属性数据类型

       例：LO是一个标准的VR表示，说明是long String在DICOM的Part5，22页，table 6.2 DICOM VALUE REPRESENTATIONS可以看到VR说明 注：VR在实际存储编码中不一定会有，这取决于采用的transfer syntax(传输语法)，采用implicit little endian的语法时，VR就是没有的，这就需要软件系统自己有套字典可以根据Tag去查找VR 。

       在DICOM Part6 ：data dictionary 可以查阅tag位及对应的VR

       3) Value Length 属性值的长度

       4) Value field 属性值

       范例：使用UltraEdit软件打开一个DICOM File来观察attributes

Pixel data介绍

       3) Pixel data is treated the same as all other attributes

        (7FE0,0010) - Pixel Data tag

        Pixel data的存储方式，包括：未压缩，压缩(无损、有损压缩)

        影像的Transfer Syntax UID (0002,0010)決定了影像的存储方式，比如：

        未压缩：

        Implicit VR Little Endian:

        Default Transfer Syntax for DICOM 　　1.2.840.10008.1.2

        Explicit VR Little Endian 　　1.2.840.10008.1.2.1

        Explicit VR Big Endian 　　1.2.840.10008.1.2.2

        无损压缩：

        JPEG Lossless : Default Transfer Syntax for Lossless JPEG

        ImageCompression 　　1.2.840.10008.1.2.4.70

        JPEG 2000 Image Compression(Lossless Only) 　　1.2.840.10008.1.2.4.90

        RLE Lossless 　　1.2.840.10008.1.2.5

        有损压缩：

        Default Transfer Syntax for Lossy

        JPEG 12 Bit Image Compression 　　1.2.840.10008.1.2.4.51

        LOSSY(8 bit) 　　1.2.840.10008.1.2.4.50

        MPEG2 image Compression 　　1.2.840.10008.1.2.4.100
